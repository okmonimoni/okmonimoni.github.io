MoniMoni

把一段自然语言剧本变成可以玩的互动模拟器。MoniMoni 会识别类型、角色、关系和规则，把长文本拆成剧场卡片、对话、按钮选项和可保存进度。

进入试玩版 看看它怎么工作
一键导入 剧本、角色、规则自动整理
按钮推进 少读设定，多做选择
长期记忆 联系人、存档、剧情进度保留

不是普通聊天框

它更像一个会读剧本的运行台。

MoniMoni 的核心不是让 AI 无边界续写，而是让 AI 先理解模拟器的玩法，再把每一次推进整理成网页能读懂、用户能点击的情节节点。

1

解析剧本

从原文里提取世界观、角色、关系、机制和选项，把松散设定变成可运行配置。

2

选择类型

恋综、生活、推理、竞演、沙盒等预设会决定每轮应该出现的模块和反馈。

3

推进剧情

情节、对话、观察室、短信、关系变化会被拆出来显示，选项变成清晰按钮。

4

保留进度

角色记忆、联系人、存档和自填模型配置都可以保留，下一次接着玩。

预设模拟器类型

每个题材，都应该有自己的玩法骨架。

同样是推进剧情，恋综要有观察室和匿名短信，推理要有线索和证词，竞演要有资源和排名。MoniMoni 让模型按类型运行，而不是每次都随意续写。

恋综

换乘观察室

前任线、匿名短信、观察室推理、约会任务和观众热度共同推动剧情。

生活

角色关系日常

固定角色群、长期记忆、好感变化和秘密累积，适合慢热陪伴型剧本。

推理

案件与线索板

区分事实、证词和推断，让每次选择都能改变调查方向。

沙盒

世界观经营探索

地点、阵营、资源和事件互相影响，适合开放世界和长期模拟。

从文本到体验

长本子不再压成一整屏文字。

AI 负责理解和推进，页面负责把可读、可点、可保存的部分呈现出来。模型和网页各做自己擅长的事。

01

导入

粘贴或选择默认模拟器，保留原始设定作为唯一源文本。

02

识别

判断题材类型，抽取角色、关系、机制、禁忌和初始选项。

03

运行

按类型预设生成场景、对话、状态反馈和下一组行动。

04

沉淀

保存联系人记忆和当前进度，让下一次推进仍然接得住。

为试玩而生，也为正式版准备

当前版本默认接入了可用模型，正式版会移除默认 key，并支持用户填写自己的模型、Base URL 和 API Key。

反馈入口会收集邮箱、Threads ID 和愿望清单，用于上线通知与产品改进。

只服务体验

导入内容用于解析、运行、生成选项和展示情节，不用于广告追踪。

本地保留

联系人、存档和自填 API 配置优先保存在浏览器本地。

可编辑记忆

自动识别到的角色可以进入通讯录，记忆内容也可以手动维护。

轻量开始

没有剧本也能从默认恋综模拟器进入，先玩起来再慢慢调整。

让模拟器不只是一段提示词。

MoniMoni 想把“读本子、理解规则、做选择、记住进度”这些麻烦事交给界面和模型，让玩家只需要进入情节。
